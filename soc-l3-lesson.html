MODULE 01 · MALWARE TRIAGE & ANALYSIS
Static vs Dynamic Analysis
🔬
STATIC ANALYSIS
Examine the file without running it. Extract strings, check imported API calls, measure section entropy. High entropy (7.9+/8.0) = packed or encrypted code.
🧪
DYNAMIC ANALYSIS
Execute in an isolated sandbox (Any.run, Cuckoo). Watch what it actually does: files created, registry keys written, network connections made.
analyst@isolated:~$ strings svchost64.exe | grep -E "(http|cmd|reg)"
http://91.215.14.22/gate.php ← C2 URL hardcoded in binary
cmd.exe /c powershell -enc %s ← encoded payload execution
HKCU\Software\Microsoft\Windows\...\Run ← persistence registry key
.text section entropy: 7.9/8.0 ← PACKED — likely UPX or custom
→ Family: AgentTesla Stealer variant
MODULE 02 · DIGITAL FORENSICS
Memory Forensics — Volatility
PROCESS MEMORY LAYOUT — WKST-011
svchost.exe
PID 664 — NORMAL
PPID: services.exe ✓
notepad.exe
PID 4412 — CHECK
Spawned from explorer.exe | RWX region detected
svchost.exe
PID 3882 — INJECTED
⚑ PPID: notepad.exe | MZ header | PAGE_EXECUTE_READWRITE
WHY THIS MATTERS
Fileless malware never writes to disk — it injects into legitimate processes in RAM. Kill the process or reboot → evidence is gone. Memory forensics is the only way to catch it.
MODULE 03 · PHISHING & EMAIL FORENSICS
Reading Email Headers
// Email header analysis — tracing the true origin
From: "IT Helpdesk" <it-support@company-helpdesk.net>
↑ TYPOSQUAT — not company.com
Received: from mail.helpdesk-company.net (185.244.31.12)
↑ Different domain again — sender is hiding true origin
X-Mailer: PHPMailer 6.0.0
↑ Bulk mail tool — not Outlook or Exchange
Subject: URGENT: Your account expires in 2 hours
↑ Artificial urgency — classic phishing pressure tactic
Domain age: 3 days | Reputation: KNOWN PHISHING INFRASTRUCTURE
Rule: Always check the sending IP, domain age, mailer, and whether the display name matches the actual domain.

MODULE 05 · THREAT HUNTING
Hypothesis-Driven Hunting
STEP 01 · HYPOTHESIZE
"An attacker has established persistence via scheduled tasks (T1053.005)." Pick a technique from ATT&CK that you have no detection coverage for.
STEP 02 · COLLECT DATA
Query all scheduled tasks across your entire endpoint fleet. Pull the full list from EDR or SIEM with timestamps and creation context.
STEP 03 · STACK COUNT
Count how many machines have each task. Tasks on many machines = normal. Tasks on 1–3 machines = investigate. Rare = suspicious.
STEP 04 · OPERATIONALIZE
Every successful hunt produces a detection rule. The next time this technique is used, the SIEM catches it automatically.
MODULE 06 · RED TEAMING FUNDAMENTALS
Red Team vs Penetration Test
PENETRATION TEST
📋Specific scope defined in advance
⏱️Time-boxed: 1–2 weeks typical
🔍Find all vulnerabilities in scope
👁️Blue team is usually notified
📄Technical report of findings
RED TEAM ENGAGEMENT
🎯Goal-based: "reach the crown jewels"
📅Longer: weeks to months
🕵️Stealth is the priority — stay hidden
🚨Blue team is NOT told — tests real response
📊Business risk report for leadership
PURPLE TEAMING: Red and blue work together in real time. Red runs a technique → blue tries to detect it → both share notes immediately. Fastest way to close detection gaps.
