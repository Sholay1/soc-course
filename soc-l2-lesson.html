MODULE 01 · BUILDING DETECTION RULES
Writing a Password Spray Detection Rule
// SPL — SPLUNK SEARCH PROCESSING LANGUAGE
index=wineventlog EventCode=4625 /* Failed logon */
| stats count dc(TargetUserName) as unique_users by src_ip
| where unique_users > 10 AND count < 50
/*
unique_users > 10 → hitting many different accounts
count < 50 → staying below lockout threshold
= Classic T1110.003 Password Spray pattern
*/
BRUTE FORCE
Many passwords × ONE user
→ Triggers lockout fast
≠
PASSWORD SPRAY
One password × MANY users
→ Stays under lockout threshold

Detecting C2 Beaconing in Wireshark
// Beacon traffic pattern — malware checking in with C2 server
// Filter: ip.dst == 91.215.14.22 && tcp.port == 443
 
No. Time Source Destination Length
1 00:00:00.0 10.0.1.55 91.215.14.22 287 ← beacon
2 00:01:00.1 10.0.1.55 91.215.14.22 291 ← beacon +60.1s
3 00:02:00.2 10.0.1.55 91.215.14.22 289 ← beacon +60.1s
→ Interval: exactly 60.0s ± 0.2s — NOT human behaviour
→ Certificate CN: *.cloudflare-analytics.net (FAKE CDN)
→ VERDICT: Malware C2 beaconing — isolate this host
KEY RULE: Legitimate user traffic is irregular. Clock-like regularity = automation = malware.

MODULE 06 · COMPLEX INCIDENT HANDLING
Ransomware Response Playbook
01
ISOLATE IMMEDIATELY
Cut the infected machine from the network the second you confirm ransomware. Every second it stays connected = more hosts encrypted.
02
PRESERVE MEMORY FIRST
Take a RAM dump before rebooting. Encryption keys live in memory. Reboot = keys gone = no decryption possible even if you pay.
03
IDENTIFY THE VARIANT
Upload a ransom note or sample file to ID Ransomware or VirusTotal. Variant determines whether free decryption tools exist.
04
NOTIFY & SCOPE
Notify CISO, Legal, and affected business units. Determine blast radius: how many hosts? Was data exfiltrated before encryption?
05
RECOVER FROM BACKUP
Verify backups are clean and were not encrypted. Restore from last known good state. Patch the entry point before reconnecting.
